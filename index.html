🚀 Guía de configuración inicial

1

Estructura del Google Sheet

El Sheet "figus" ya tiene la hoja Stickers con el catálogo completo.

Columnas de Stickers:
A: code · B: country · C: country_code · D: number · E: description · F: type · G: category · H: is_official · I: created_at

La app usa el code como identificador único de cada figurita.

Album (estado del álbum familiar): code | estado | cantidad_rep
Estados válidos: missing | owned | repeated

Intercambios (historial): fecha | de | para | da | recibe | notas

2

Obtener el Sheet ID

El ID está en la URL de tu planilla: docs.google.com/spreadsheets/d/ESTE_ID/edit
Copialo y pegalo en CONFIG.SHEET_ID en el archivo index.html.

3

Crear proyecto en Google Cloud

Entrá a console.cloud.google.com → Nuevo proyecto → Habilitar APIs:
Google Sheets API y Google Drive API

APIs & Services → Credentials → Create OAuth 2.0 Client ID
Tipo: Web application
Authorized JavaScript origins: http://localhost (dev) y tu URL de GitHub Pages

Copiá el Client ID y pegalo en CONFIG.CLIENT_ID.

4

Compartir el Google Sheet

Compartí la planilla con los 4 integrantes de la familia con rol Editor.
Solo pueden editar quienes tengan acceso al Sheet.

5

Deploy en GitHub Pages

Subí el archivo index.html a un repo de GitHub.
Settings → Pages → Branch: main → carpeta: / (root)
La URL será: https://tuusuario.github.io/tu-repo/

Agregá esa URL a los Authorized origins en Google Cloud Console.

6

Invitar a la familia

Compartí la URL de GitHub Pages con los 4 integrantes. Cada uno inicia sesión con su cuenta Google y puede ver/editar el álbum.
